> 最后更新：2026-06-26

## 当前进度总览

### 已实现的核心能力

1. **声明式 DSL v1 + v2** -- 完整的组件系统和多Agent编排语法
2. **交互式 REPL + 流式输出** -- prompt_toolkit 全屏 TUI + 降级模式双通道
3. **多 Provider 支持** -- DeepSeek/OpenAI/Anthropic/Ollama，opencode 风格配置
4. **工具系统** -- shell/read/write/glob/grep 内置工具 + MCP 集成 + 统一注册表
5. **安全机制** -- 权限控制(allow/deny/ask) + 熔断器 + Doom Loop 检测
6. **Agent Loop** -- 完整的 Agent 主循环，支持工具调用和流式输出
7. **会话管理** -- SQLite 持久化 + 断点恢复 + 上下文压缩
8. **AI 辅助** -- 工作流自动生成 + 上下文压缩 + 流式处理
9. **工作流编辑器** -- Textual TUI 交互式编辑器 + htop 风格监控面板

## 已知问题

### 从诊断报告提取

1. **流式输出在非降级模式下的稳定性** -- 已通过多轮修复（`7e11d18`, `b1ada6d`, `7e84e8e`），当前方案使用 `call_soon_threadsafe` 从 Agent Loop 线程通知主线程消费 UI 更新队列，但需要更多场景测试

## 与竞品的差异化

| 维度 | Langflow | n8n | CrewAI | **GrassFlow** |
|------|----------|-----|--------|---------------|
| 定位 | 全栈AI平台 | 业务自动化 | 代码框架 | 轻量纯编排 |
| 交互方式 | GUI | GUI | 代码 | GUI + TUI |
| 依赖声明 | 手动连线 | 手动连线 | 代码 | 声明式 DSL |
| 交互类型 | 线性 | 线性+条件 | 线性 | 线性+条件+广播+聚合+立即执行 |
| AI生成编排 | 不支持 | 不支持 | 不支持 | 支持 (TUI) |
| 监控Agent | 不支持 | 不支持 | 不支持 | 支持 |
| 分发方式 | pip/Docker | npm/Docker | pip | pip (TUI) + exe (GUI) |

## 开发规范

- Python 环境：使用项目目录下的虚拟环境 `.venv`
- 修改前确保 git 仓库干净，修改后立即提交
- Windows 环境优先，需要 Linux 命令时使用 `wsl -d kali-linux -- <command>`
- 输出和思考使用中文
- 遵循 CLAUDE.md 中的项目规则
